*Ejemplos aplicados a su proyecto* 
-
Este documento presenta el análisis y modelamiento de procesos utilizando la notación BPMN (Business Process Model and Notation) aplicados al proyecto del SENA denominado:

“Implementación de un Sistema de Gestión de Proyectos para la Operación de una Lavandería”

El objetivo es demostrar el uso de técnicas de gestión de procesos en el marco del ciclo de vida de un proyecto, utilizando diagramación conceptual, eventos, roles y flujos propios de BPMN.

Este documento sirve de soporte teórico y práctico para las fases:

Formulación

Planeación

Ejecución

Control

Cierre

2. Descripción del Proyecto

El proyecto busca optimizar y estandarizar los procesos de una lavandería mediante la implementación de un sistema básico de gestión, donde se identifiquen, modelen y mejoren los flujos de trabajo principales.

Objetivo General

Implementar un sistema de gestión de procesos basado en BPMN que permita mejorar la eficiencia, el tiempo de entrega y el control operativo de una lavandería.

Objetivos Específicos

Identificar los procesos actuales.

Modelar el proceso principal utilizando BPMN.

Definir roles, actividades, eventos y puntos de decisión.

Proponer mejoras a partir del análisis del flujo.

Integrar el modelo dentro del plan de gestión del proyecto.

3. Identificación de Procesos (Visión BPMN)

Los procesos principales de la lavandería, y que serán modelados en el proyecto del SENA, son:

Recepción del Cliente

Clasificación de Prendas

Lavado

Secado

Planchado y Acabados

Control de Calidad

Entrega al Cliente

En BPMN, estos procesos se representarían dentro de una piscina, con carriles asociados a:

Cliente

Recepcionista

Operario de lavado

Operario de planchado

Calidad

Administración

4. Aplicación de BPMN al Proyecto (Ejemplos Prácticos)

A continuación se presentan modelos textuales de BPMN (explicados paso a paso), que simulan los diagramas que se elaborarían en Bizagi, Draw.io o Visio.
Estos son totalmente válidos como soporte documental.

4.1. Ejemplo BPMN – Proceso de Inicio del Proyecto
Evento de Inicio

Start Event: Solicitud del administrador para optimizar procesos.

Actividades

Analizar procesos actuales
Rol: Líder de proyecto

Reunir actores clave (stakeholders)
Rol: Administrador

Crear Acta de Constitución

Aprobación del Acta (Gateway)

Gateway de Decisión

¿El acta está aprobada?

Sí: Continuar con la planeación.

No: Regresar a revisión.

Evento de Fin

Acta aprobada y proyecto iniciado.

4.2. Ejemplo BPMN – Planeación del Proyecto
Secuencia BPMN Detallada

Start Event: Inicio de la planeación

Identificar procesos a modelar

Reunir información operativa

Diseñar el modelo BPMN preliminar

Revisión del modelo (Gateway)

¿Cumple los estándares del SENA / PMI?

Sí → Continuar

No → Ajustar

Generar el Plan de Gestión del Proyecto

End Event: Planeación aprobada

4.3. Ejemplo BPMN – Ejecución (Modelado del Proceso Principal)
Start Event: Procesos listos para modelar.

Lane 1 – Recepcionista

Recibir prendas

Registrar información en el sistema

Generar orden de servicio

Lane 2 – Operario de Clasificación

Revisar tipo de prenda

Realizar tratamiento previo

Lane 3 – Operario de Lavado

Seleccionar ciclo

Ejecutar lavado

Revisar limpieza

Gateway: ¿Requiere reproceso?

Sí → Volver al operario

No → Continuar

Lane 4 – Operario de Secado

Secado mecánico o al aire

Lane 5 – Planchado

Planchado / doblado

Preparación del empaque

Lane 6 – Calidad

Revisión final

Autorización

Lane 7 – Cliente

Recoger prendas

Cerrar pedido

End Event: Servicio completado.
4.4. Ejemplo BPMN – Control del Proyecto
Flujo BPMN textual:

Start Event: Avances registrados

Comparar avance real vs planificado

Evaluar tiempos del proceso BPMN

Identificar cuellos de botella

Generar informes

Reunión semanal de ajustes

Gateway: ¿Se requiere acción correctiva?

Sí → Implementar mejoras

No → Continuar

End Event: Control aplicado

4.5. Ejemplo BPMN – Cierre del Proyecto
Actividades

Documentar el proceso final

Registrar lecciones aprendidas

Capacitar al personal

Entregar modelo BPMN final

Cerrar el proyecto ante la administración

End Event

Proyecto finalizado y entregado.

5. Integración del BPMN con la Gestión de Proyectos del SENA

El proyecto combina:

Área SENA / PMI	Cómo se aplica con BPMN
Alcance	Los procesos modelados determinan los entregables.
Tiempo	Los eventos y actividades BPMN sirven como base del cronograma.
Recursos	Cada lane refleja roles y actores involucrados.
Riesgos	Los gateways permiten identificar puntos críticos.
Calidad	Se modela el proceso de control.
Comunicaciones	El diagrama BPMN permite al equipo entender el flujo.
6. Propuesta de Mejora Basada en BPMN

Tras analizar el proceso modelado, se identifican mejoras:

✔ Implementar un sistema de etiquetado digital

Mejora el control del proceso y la trazabilidad.

✔ Reducción de reprocesos

Al identificar puntos críticos en los gateways.

✔ Estandarización de tiempos

Los eventos intermedios ayudan a medir cuánto dura cada etapa.

✔ Mayor claridad en roles

Los lanes reducen confusiones operativas.

7. Conclusión

La aplicación de BPMN dentro de un proyecto del SENA permite:

Modelar procesos de manera clara.

Gestionar mejor los recursos y tiempos.

Identificar cuellos de botella en el flujo de servicio.

Optimizar las operaciones de la lavandería.

Fortalecer todas las áreas de conocimiento de la Gestión de Proyectos.

Este documento proporciona el soporte teórico, técnico y práctico para la evidencia requerida por el SENA.
